# Recipe IDs

Complete list of recipe IDs generated by Nostalgic Trees. Use these with `event.remove({ id: ... })` in KubeJS scripts.

## Recipe type IDs

Used with `event.remove({ type: ... })` to target all recipes of a type, or with `event.custom({ type: ... })` to add new ones.

| Type ID | Purpose |
|---|---|
| `nostalgictrees:mutation` | Sapling mutation via bee pollination |
| `nostalgictrees:drying` | Drying rack transformations |

All other NT recipes use vanilla types: `minecraft:crafting_shaped`, `minecraft:crafting_shapeless`, `minecraft:smelting`, `minecraft:smithing_transform`.

## Per-tree recipe IDs

For every tree in `config/nostalgictrees/trees/`, two recipes are generated. Replace `<tree>` with the tree's `name` value (`iron`, `diamond`, `copper`, etc.).

| ID pattern | Type | What it does |
|---|---|---|
| `nostalgictrees:<tree>_chunk` | `crafting_shapeless` | Stripped log → chunk |
| `nostalgictrees:<tree>_resource` | `crafting_shaped` | 4 apples + 4 chunks + 1 honeycomb → tree's output item |

## RGB dye conversion IDs

One recipe per vanilla dye color. Pattern: `nostalgictrees:rgb_dye_to_<color>`.

| ID pattern | Example |
|---|---|
| `nostalgictrees:rgb_dye_to_<color>` | `nostalgictrees:rgb_dye_to_white`, `nostalgictrees:rgb_dye_to_orange`, etc. |

Where `<color>` is any of: `white`, `orange`, `magenta`, `light_blue`, `yellow`, `lime`, `pink`, `gray`, `light_gray`, `cyan`, `purple`, `blue`, `brown`, `green`, `red`, `black`.

## Discovering IDs yourself

If something isn't listed here (new trees, unreleased features), dump the registry:

1. In-game, run `/kubejs dump`
2. Open `kubejs/exported/recipes.txt`
3. Search for `nostalgictrees:` — every NT recipe ID is there

This is also the fastest way to confirm your `event.remove` target actually exists. A typo in the ID means `event.remove` silently does nothing — checking the dump first saves confusion.
